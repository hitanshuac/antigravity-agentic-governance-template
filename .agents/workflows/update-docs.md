# Update Docs Workflow

<workflow_steps>
1. Read the current contents of `src/capabilities/` and any active application source files.
2. Analyze the latest codebase changes.
3. Update `README.md` to reflect the latest APIs, database architecture, and dependencies.
4. Update any other relevant `.md` files (like `HANDOVER.md` or `retrospective.md`) to document the current system state.
5. Create a `walkthrough.md` artifact summarizing the documentation changes.
6. Ask the user to review the updated markdown files.
